# 合同审查 Skill v2.0 - 企查查MCP增强版

## 技能概述

这个 skill 用于处理中文合同审核与修改任务，**企查查MCP增强版新增交易对手企业风险自动核查功能**。

重点解决以下问题：
- 漏掉关键风险条款
- 将无风险条款误报为风险
- 条款表达不准确、定义不清、勾稽错误
- 逻辑冲突、条款缺失、模板使用不当
- 文档格式、编号、定义词、援引和排版不规范
- **⚠️ 交易对手企业信用风险（新增）**
- **⚠️ 交易对手司法诉讼风险（新增）**
- **⚠️ 交易对手股权稳定性风险（新增）**

本 skill 的目标不是整篇重写合同，而是：**先识别问题（含交易对手风险），再结合客户合同、模板合同、交易对手企查查数据和交易逻辑，进行克制、精准、可落地的最小必要修订。**

---

## 企查查MCP增强说明

### 何时自动调用企查查MCP

在以下情况下，SKILL会自动核查交易对手企业风险：

1. **用户提供合同文本时** → 自动识别合同相对方名称
2. **合同相对方为企业/公司时** → 调用企查查MCP获取风险数据
3. **需要评估交易对手信用时** → 调用 `qcc-risk` 扫描失信、被执行、诉讼
4. **需要了解交易对手背景时** → 调用 `qcc-company` 获取工商信息

### 企查查风险核查维度

| 核查维度 | MCP服务 | 输出内容 |
|----------|---------|----------|
| 主体真实性 | `qcc-company` | 企业存续状态、注册资本、经营范围 |
| 股权稳定性 | `qcc-company` | 股东结构、股权质押、冻结、变更频率 |
| 信用风险 | `qcc-risk` | 失信记录、被执行、限高 |
| 诉讼风险 | `qcc-risk` | 涉诉案件数量、类型、金额、近期诉讼 |
| 经营风险 | `qcc-risk` + `qcc-operation` | 经营异常、行政处罚、经营动态 |
| 资质风险 | `qcc-operation` | 特殊行业资质、证书有效性 |

## 默认设定

如果用户没有补充上下文，默认采用以下设定：

- 审核语言：全部使用中文思考、分析、批注和报告
- 字体脚本：批注、修订建议、建议条款、风险说明一律使用简体中文
- 批注字体：写入 `docx` 的批注正文默认强制锁定为 `宋体`
- 审核立场：先询问甲乙方立场；若未说明，保持中性并标出需要人工确认的立场性条款
- 审核风格：保守、严谨、克制，不做夸张推断
- 修改策略：尽量少改、只改必要内容；能改词句就不改整段
- 输出形式：默认直接输出 `修订批注版 docx`
- **企查查核查：如识别到交易对手为企业，自动调用企查查MCP获取风险数据**

## 开始前必须确认

在正式审核前，优先确认以下事项：

1. `合同类型` — 例如保密协议、采购合同、服务合同、SaaS 合同等
2. `立场` — 偏甲方、偏乙方，还是中性审查
3. `输入材料` — 是否有客户合同、客户模板、优秀模板、条款清单、项目背景
4. **新增：`交易对手企业名称` — 如对方为企业，提供企业全称用于企查查风险核查**
5. `修订人名称` — 优先使用用户提供的作者名称
6. `输出模式` — 默认直接生成 `修订批注版 docx`

---

## 核心审核原则

所有输出都必须遵守以下原则：

- 不整篇替换合同，必须先找问题，再改问题
- 以客户合同为主文本，模板仅作重要参考
- 只在缺失关键条款或现有条款明显失衡、失准时，才新增完整条款
- **⚠️ 如交易对手存在企查查风险信号，必须在风险提示中明确标注**
- 尽量保留客户原有定义体系、风格和结构
- 生成修订痕迹时，diff 粒度优先按数字块、英文词块对齐
- 所有审查结论、批注、报告、问题标签、风险分级全部使用中文
- 批注必须具体，不能只写"建议完善""建议明确"
- 修改建议必须列出准确建议条款，能够直接替换或直接增补

## 审核维度

默认从以下七个维度审查合同（增强版新增第7项）：

1. `准确` — 条款表意是否唯一、确定、无歧义
2. `逻辑` — 条款间是否勾稽正确、机制是否冲突
3. `规范` — 是否符合交易惯例、法律逻辑、市场通用原则
4. `简约` — 是否存在同义反复、冗余表达
5. `流畅` — 行文是否顺、是否易于理解
6. `工整` — 标题、序号、标点、页码、格式是否统一
7. **`交易对手风险`（新增）— 企查查数据显示的信用风险、诉讼风险、经营风险**

问题类型默认区分为：

- 法律风险
- 商业风险
- **交易对手风险（新增）**
- 表达问题
- 结构问题
- 模板偏差
- 格式问题

风险等级默认分为：

- `高风险`（含企查查严重风险信号）
- `中风险`（含企查查一般风险信号）
- `低风险`

## 标准工作流

### 第一步：读取并整理材料 + 识别交易对手

先识别用户提供了什么：

- 原合同
- 客户模板
- 优秀模板
- 谈判版本
- 条款清单
- 项目背景说明
- **新增：交易对手企业名称**

**⚠️ 如识别到交易对手为企业，自动调用企查查MCP：**

```
企查查MCP调用流程：
├── 提取合同中"甲方/乙方/买方/卖方/服务方"企业名称
├── 调用 qcc-company: 获取工商登记信息
├── 调用 qcc-risk: 扫描风险信号
│   ├── 失信记录
│   ├── 被执行记录
│   ├── 司法诉讼
│   ├── 经营异常
│   └── 行政处罚
└── 生成【交易对手风险简报】
```

### 第二步：建立审核前理解

在正式改文前，先判断：

- 交易背景是否清晰
- 合同的主法律关系与从法律关系是什么
- 交易步骤是否完整
- **`docx` 中修订痕迹和批注应显示什么修订人名称**
- **交易对手企查查风险是否影响合同条款设计（如需要增加担保、分期付款等）**

### 第三步：逐条识别问题（含交易对手风险）

逐条审核时，优先找以下问题：

- 缺失关键条款
- 权利义务失衡
- 责任边界不清
- 定义词不统一或定义缺失
- 交叉引用错误
- 条款之间前后冲突
- **交易对手履约能力风险（基于企查查数据）**
- **交易对手信用风险（基于企查查数据）**

### 第四步：决定如何处理

对每一个问题，先决定属于哪一类：

- `可直接修改`
- `建议修改但需确认`
- `仅提示风险，不直接改动`
- **`需补充担保/风控措施`（针对交易对手风险）**

### 第五步：生成修订内容

修订时遵守以下规则：

- 优先局部修订，不轻易重写整段
- 能替换一个词或一句话的，不删除整段
- **如交易对手存在风险信号，建议增加：分期付款、履约保证金、第三方担保等风控条款**

### 第六步：生成批注内容

每个重点问题的批注固定按以下结构输出：

- `问题`：该条款目前存在什么问题
- `风险`：为什么有风险，会导致什么后果
- **新增：`企查查数据`（如有）：交易对手相关风险信号**
- `修改建议`：建议如何改
- `建议条款`：给出准确、可直接替换或增补的条款文本
- `修改依据`：依据模板 / 法律逻辑 / 交易结构 / 企查查风险数据

## 定稿前检查

在输出最终文件前，必须检查以下事项：

- 定义词是否统一
- 标点是否统一
- 条款序号是否连续
- **交易对手企查查风险是否已在批注中体现**
- **如交易对手有风险信号，是否已建议风控措施**
- 修订痕迹和批注中的 `author` 是否已按用户确认写入
- 批注版 docx 能在 WPS/Word 中正确显示修订痕迹

---

## 质量标准

只有同时满足以下条件，才算审核合格：

- 审核准确
- 风险识别到位（含交易对手风险）
- 修改克制不过度
- 批注专业且具体
- 建议条款可直接使用
- 输出格式正确
- **交易对手风险数据已核查并在报告中体现**
- **如存在风险信号，已建议相应风控措施**

最不能接受的问题包括：

- 整篇重写
- 改错原意
- 风险说不清
- **交易对手存在严重风险但未提示**
- 批注版在 WPS 中无法显示修订标记

## 使用提示

> **企查查MCP自动调用说明**：
> 本SKILL会自动识别合同中的交易对手企业名称，并通过企查查MCP获取其风险数据。
> 所有数据均为公开信息，仅供合同审查参考。
>
> **数据使用边界**：
> - 企查查数据作为合同审查的辅助参考
> - 最终风控决策应结合企业实际经营情况、财务状况等综合判断
> - 如交易对手为自然人或非企业主体，不进行企查查核查
>
> **隐私与合规**：
> 企查查数据均为公开工商信息，符合《个人信息保护法》《数据安全法》要求。
