# 法律尽职调查 Skill v2.0 - 企查查MCP增强版

## 目标

为中国律师提供系统化的、数据驱动的法律尽职调查底稿编写与报告生成工具。

**核心价值**（增强版新增企查查数据自动化）：
- **调查全面**：10大标准板块、200+检查项，不遗漏重要事项
- **数据驱动**：企查查MCP自动获取企业工商、股权、风险、知产数据
- **用语规范**：标准法律用语，表述专业严谨
- **两阶段输出**：底稿（律师内部记录）→ 报告（客户交付文件）
- **风险识别**：三级风险评估，系统识别法律风险

---

## 工作流程

```
init 初始化  →  [QCC数据获取]  →  draft 写底稿  →  check 检查  →  report 生成报告
（创建项目）    （企查查MCP自动）   （数据预填充）     （完整性）       （底稿→报告）
                     ↺ 循环
```

## 企查查MCP增强说明

### 何时自动调用企查查MCP

在以下情况下，SKILL会自动通过MCP调用企查查数据服务：

1. **用户提供目标公司名称时** → 自动获取企业全景数据
2. **需要核验工商登记信息时** → 调用 `qcc-company`
3. **需要分析股权结构时** → 调用 `qcc-company` 进行股权穿透
4. **需要识别风险信号时** → 调用 `qcc-risk` 扫描18类风险
5. **需要盘点知产资产时** → 调用 `qcc-ipr` 获取专利商标软著
6. **需要了解经营状况时** → 调用 `qcc-operation` 获取招投标资质

### MCP调用与尽调章节的对应关系

| 尽调章节 | 企查查MCP服务 | 自动获取数据 |
|----------|--------------|--------------|
| 第1章 主体资格 | `qcc-company` | 工商登记、企业状态、经营范围 |
| 第2章 股权结构 | `qcc-company` | 股东信息、股权穿透、历史变更 |
| 第3章 公司治理 | `qcc-company` | 主要人员、关联企业 |
| 第4章 核心资产 | `qcc-ipr` + `qcc-operation` | 专利商标软著、资质证书 |
| 第5章 业务经营 | `qcc-operation` | 招投标、经营动态 |
| 第8章 债权债务 | `qcc-risk` | 股权质押、动产抵押 |
| 第9章 诉讼仲裁 | `qcc-risk` | 司法诉讼、失信被执行、开庭公告 |

## 四种工作模式

### 模式 1：init（项目初始化 + 企查查数据预获取）

**触发词**：`初始化尽调项目`、`新建尽调`、`init DD`、`对XX公司进行尽调`

**操作步骤**：
1. 获取目标公司全称（用户提供）
2. **自动调用企查查MCP**：
   - `qcc-company`: 获取工商登记、股东结构、主要人员
   - `qcc-risk`: 扫描风险信号（失信、被执行、诉讼等）
   - `qcc-ipr`: 获取知识产权清单
   - `qcc-operation`: 获取经营资质、招投标记录
3. 运行 `scripts/init_project.py`，在指定路径创建项目目录
4. 生成项目信息文件 `project-info.md`（含企查查数据摘要）
5. 为10个章节各生成底稿模板文件（企查查数据已预填充至相关章节）

**必须获取的输入**：
- `project_path`：项目存放路径
- `target_company`：目标公司全称（**用于企查查MCP查询**）
- `client_name`：委托人名称

**可选输入**（有默认值）：
- `base_date`：调查基准日（默认当日）
- `purpose`：调查目的（如"股权收购"、"投资入股"）
- `law_firm`：律师事务所名称
- `lawyers`：经办律师姓名（可多个）

**输出**：项目目录结构如下：

```
[项目名称]-DD/
├── project-info.md          # 项目基本信息与进度跟踪（含企查查数据摘要）
├── working-paper.md         # 完整底稿（10章合一，单一文件）
│                            # 【企查查数据已预填充】
├── qcc-data/                # 企查查原始数据（JSON格式，供核实）
│   ├── company-profile.json
│   ├── shareholders.json
│   ├── risk-signals.json
│   ├── ipr-assets.json
│   └── operation-records.json
└── report/
    └── （报告生成后存放于此）
```

### 模式 2：draft（撰写底稿 - 企查查数据增强）

**触发词**：`写底稿`、`撰写第X章`、`draft`、`分析材料并写入底稿`

**操作步骤**：
1. 读取 [references/section-guide.md](references/section-guide.md) 中对应章节的调查指南
2. **如已存在企查查数据，自动加载至底稿**
3. 阅读用户提供的补充调查材料
4. 按底稿六段结构撰写该章节内容（企查查数据作为§4调查发现的基础）
5. 写入项目根目录的 `working-paper.md`

**企查查数据在各章节的预填充位置**：

**第1章 公司基本信息与主体资格**：
```markdown
### 1.4 调查发现（企查查数据预填充）

【企查查MCP自动获取 - 工商登记信息】
- 企业名称：[企查查返回]
- 统一社会信用代码：[企查查返回]
- 企业类型：[企查查返回]
- 注册资本：[企查查返回]（实缴：[企查查返回]）
- 成立日期：[企查查返回]
- 法定代表人：[企查查返回]
- 经营状态：[企查查返回]
- 注册地址：[企查查返回]
- 经营范围：[企查查返回]
- 营业期限：[企查查返回]
```

**第2章 股权结构与股东信息**：
```markdown
### 2.4 调查发现（企查查数据预填充）

【企查查MCP自动获取 - 股权结构】
- 当前股东及持股比例：
  * [股东A]：[X]%
  * [股东B]：[X]%
- 股权穿透分析（如MCP支持）：
  * 【注意】企查查MCP暂不支持自动识别受益所有人/实控人
  * 该部分需通过企业提供的股东协议、投资协议等文件核实
- 历史股权变更：[X]条记录（详见企查查数据文件）
- 对外投资：[X]家关联企业
```

**第9章 诉讼仲裁与行政处罚**：
```markdown
### 9.4 调查发现（企查查数据预填充）

【企查查MCP自动获取 - 司法风险】
- 司法诉讼：共[X]条
  * 作为原告：[X]条
  * 作为被告：[X]条
- 失信记录：[有/无]
- 被执行记录：[有/无]（金额：[X]万元）
- 限制高消费：[有/无]
- 开庭公告：[X]条
- 裁判文书：[X]份

【企查查MCP自动获取 - 行政处罚】
- 行政处罚记录：[X]条
```

**必须获取的输入**：
- `chapter`：章节编号（1–10）或章节名称
- `materials`：补充调查材料（文件路径列表 或 粘贴的文本内容）

**底稿六段结构**（每章必须包含）：

```markdown
## [编号]. [章节名称]

### [X].1 调查范围与方法
调查范围：[本章调查覆盖的事项]
调查方法：
- [ ] 企查查MCP数据获取
- [ ] [方法2]

### [X].2 已获取材料清单
| 序号 | 材料名称 | 形式 | 日期/期间 | 备注 |
|------|----------|------|-----------|------|
| 1 | 企查查企业信息报告 | 电子数据 | [日期] | MCP自动获取 |

### [X].3 未获取/待补充材料
| 序号 | 材料名称 | 重要程度 | 状态 | 影响说明 |
|------|----------|----------|------|----------|

### [X].4 调查发现
[含企查查数据的事实记录，引用具体文件，不含意见判断]

### [X].5 风险提示
| 序号 | 风险事项 | 等级 | 事实依据 | 法律依据 | 处理建议 |
|------|----------|------|----------|----------|----------|

### [X].6 律师备忘
[内部工作记录，不进报告]
```

**风险等级标准**：
- 🔴 高：直接影响交易可行性或重大法律后果
- 🟡 中：需关注，可能影响交易条款或需要整改
- 🟢 低：轻微瑕疵，一般不影响交易，建议关注

### 模式 3：check（检查底稿）

**触发词**：`检查底稿`、`完整性检查`、`check`、`汇总风险`

**操作步骤**：
1. 读取 `working-paper.md` 全文
2. 逐章检查六段结构完整性
3. **核对企查查数据与底稿填写的一致性**
4. 汇总所有风险提示（按等级分类）
5. 统计未获取材料
6. 输出检查报告到对话（不修改底稿）

**输出格式**：

```
## 底稿完整性检查报告

### 企查查数据使用情况
✅ 第1章 主体资格：已填充企查查工商数据
✅ 第2章 股权结构：已填充企查查股东数据
✅ 第9章 诉讼仲裁：已填充企查查司法风险数据

### 各章完成情况
| 章节 | 调查范围 | 材料清单 | 调查发现 | 风险提示 | 状态 |
...

### 风险汇总
🔴 高风险（X项）：...
🟡 中风险（X项）：...
🟢 低风险（X项）：...

### 待补充材料（X项）
...
```

## 企查查MCP数据字段说明

### qcc-company 返回字段
- `companyName`: 企业名称
- `creditCode`: 统一社会信用代码
- `companyType`: 企业类型
- `registCapi`: 注册资本
- `realCapi`: 实缴资本
- `startDate`: 成立日期
- `operatingStatus`: 经营状态
- `legalPerson`: 法定代表人
- `registAddress`: 注册地址
- `scope`: 经营范围
- `shareholders`: 股东列表
- `history`: 历史变更记录
- `investments`: 对外投资

### qcc-risk 返回字段
- `dishonestList`: 失信记录
- `executeList`: 被执行记录
- `limitHighConsumption`: 限高记录
- `judicialCase`: 司法诉讼
- `courtNotice`: 开庭公告
- `punishList`: 行政处罚
- `abnormal`: 经营异常
- `stockFreeze`: 股权冻结
- `mortgage`: 动产抵押

### qcc-ipr 返回字段
- `patents`: 专利列表
- `trademarks`: 商标列表
- `softwareCopyrights`: 软件著作权

### qcc-operation 返回字段
- `bidding`: 招投标记录
- `certificates`: 资质证书
- `creditRating`: 信用评级
- `news`: 经营动态

## 使用提示

> **企查查MCP自动调用说明**：
> 本SKILL通过配置 `.mcp.json` 自动连接企查查数据服务。当SKILL识别到需要企业信息时，
> 会自动调用对应的MCP服务获取数据。所有MCP调用均为只读查询，不会修改企查查数据库。
>
> **数据准确性提示**：
> 企查查数据仅供参考，最终法律意见应基于官方工商档案、企业提供的原始文件等综合判断。
>
> **隐私与合规**：
> 企查查数据均为公开信息，符合《个人信息保护法》《数据安全法》要求。
